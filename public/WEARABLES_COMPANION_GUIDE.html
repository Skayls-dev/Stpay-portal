ST Pay – Guide Companion Apps & Wearables

Ce guide explique comment intégrer ST Pay sur smartwatch via une app compagnon mobile, avec des scénarios détaillés, diagrammes de séquence et exemples de code pour Android et iOS.

Scénario 1 : Paiement initié depuis la montre

  1. L’utilisateur lance un paiement sur la montre.
  2. La montre transmet la demande à l’app mobile (DataLayer/WatchConnectivity).
  3. L’app mobile appelle l’API ST Pay (POST /api/Payment) avec la clé API.
  4. L’app mobile transmet le statut à la montre (QR, code, attente…).
  5. À la fin, la montre affiche le statut final.
sequenceDiagram participant Watch as Montre connectée participant Mobile as App mobile compagnon participant API as API ST Pay Watch->>Mobile: Demande de paiement (montant, ref...) Mobile->>API: POST /api/Payment (clé API) API-->>Mobile: Statut paiement (en attente) Mobile-->>Watch: Affiche statut (QR, code, attente) Note over Watch,Mobile: La montre ne détient jamais la clé API API-->>Mobile: Statut final (succès/échec) Mobile-->>Watch: Affiche statut final

Scénario 2 : Synchronisation du statut paiement

sequenceDiagram participant Watch as Montre connectée participant Mobile as App mobile compagnon participant API as API ST Pay loop Synchronisation statut API-->>Mobile: Statut paiement (polling ou push) Mobile-->>Watch: Statut mis à jour (succès, échec, attente) end Note over Watch,Mobile: La montre ne contacte jamais l’API directement

Scénario 3 : Annulation depuis la montre

  1. L’utilisateur appuie sur “Annuler” sur la montre.
  2. La montre envoie la commande à l’app mobile.
  3. L’app mobile appelle DELETE /api/Payment/{id} à l’API ST Pay.
  4. L’app mobile transmet le statut d’annulation à la montre.

Exemples de code

Android (Wear OS + Mobile)

// Sur la montre (Wear OS) — Kotlin
val request = "PAYMENT_REQUEST|amount=5000;ref=ORDER123"
val nodeId = ... // ID du mobile appairé
Wearable.getMessageClient(context)
    .sendMessage(nodeId, "/stpay/payment", request.toByteArray())

// Sur le mobile (Android) — Kotlin
override fun onMessageReceived(event: MessageEvent) {
    if (event.path == "/stpay/payment") {
        val data = String(event.data)
        // Parse et appelle l’API ST Pay
        val response = apiClient.processPayment(/* ... */)
        // Renvoie le statut à la montre
        sendStatusToWatch(response.status)
    }
}

// Appel API ST Pay (mobile)
val response = apiClient.processPayment(
    amount = 5000,

    // ...autres champs requis
)

// Transmission du statut à la montre

    .sendMessage(nodeId, "/stpay/status", statusMsg.toByteArray())

WCSession.default.sendMessage(message, replyHandler: nil, errorHandler: nil)

// Sur le mobile (iOS)

Conseils pour développeurs juniors :
  • Ne jamais stocker la clé API sur la montre : toujours côté mobile.
  • Bien gérer les erreurs : perte de connexion, refus API, etc.
  • Tester la communication montre ↔ mobile avant l’API.
  • Utiliser les snippets fournis pour chaque plateforme.
